# 하나은행 CMS 통합관제 및 데이터 대시보드

## 프로젝트 개요

하나은행 기업고객 대상 **CMS(Cash Management Service) 통합관제 시스템**입니다.  
고객사에 설치된 Agent 단말의 상태를 실시간으로 감시하고, 수집된 금융 데이터를 분석하여 영업/마케팅 제안까지 자동화하는 올인원 대시보드입니다.

> 현재는 **데모(프로토타입)** 버전으로, 모든 데이터는 `db.js`에서 인메모리로 생성됩니다.  
> 향후 실제 DB 연동 시 `db.js` 내부만 API 호출로 교체하면 됩니다.

---

## 폴더 구조

```
project/
├── index.html          ← 메인 HTML (사이드바, 헤더, 뷰 컨테이너)
└── js/
    ├── db.js           ← 데이터 레이어 (데이터 생성 + 쿼리 메서드)
    ├── dashboard.js    ← 시스템 관제 대시보드 총괄표
    ├── control.js      ← 시스템 관제 하위 화면들 (단말, Flag, 재구동 등)
    ├── data.js         ← 데이터 활용 + 영업 자동화 화면들
    └── app.js          ← 앱 코어 (라우팅, 모달, Customer360, 초기화)
```

---

## 실행 방법

1. 위 폴더 구조 그대로 로컬에 저장합니다.
2. `index.html`을 브라우저에서 열면 바로 동작합니다.
3. 별도의 서버나 빌드 과정이 필요 없습니다.

CDN으로 로드하는 외부 라이브러리: Tailwind CSS, Chart.js, Font Awesome, Pretendard(한글 폰트)

## 주요 기능

### 1. Customer 360 심층 분석 (메뉴별 맞춤)

어떤 화면에서든 **고객사 이름을 클릭**하면 오른쪽에서 심층 분석 패널이 슬라이드됩니다.  
현재 보고 있는 메뉴에 따라 분석 내용이 다르게 구성됩니다.

| 현재 메뉴 | 360 분석 내용 |
|-----------|--------------|
| 금융자산/예적금 | 은행별 보유 분포 도넛 + 만기 D-Day별 바차트 + 보유 예적금 목록 |
| B2B/전자어음 | 만기 분포 바차트 + 거래유형 도넛 + B2B 내역 테이블 |
| 법인카드 | 업종별 결제 도넛 + 월별 추이 라인 + 결제 내역 |
| 세금계산서 | 매출/매입 KPI + 월별 비교 바 + 품목별 도넛 + 거래내역 |
| 자금흐름 | 30일 입출금 차트 + 총입금/출금/순흐름 KPI |
| 시스템 관제 | 인프라 레이더 + 60일 가동 히트맵 + 장애 추이 |

각 context별로 **AI 추천 액션 문구**와 **맞춤 제안 발송 버튼**이 자동 생성됩니다.

### 2. 모니터링 모달 (Agent 재구동 / Ping 테스트)

단말 상태 관제에서 고객사 클릭 시 **Agent 상세 정보 모달**이 열립니다.

- **Agent 재구동**: 클릭 → 로딩("Watcher를 통해 재구동 명령 전송 중...") → 2초 후 성공/실패 결과 표시
- **Ping 테스트**: 클릭 → 로딩("Watcher → 스케줄 PC에서 Ping 진행 중") → 1.8초 후 "Ping 정상(time=23ms, 손실 0%)" 또는 "연결 안됨(손실 100%)" 결과 표시

실제 CMD 결과처럼 IP, 응답시간, TTL, 패킷 손실률이 표시됩니다.

### 3. 단말 상태별 맞춤 조치 버튼

단말 상태 관제 테이블의 맨 오른쪽 조치 버튼이 상태에 따라 다르게 표시됩니다.

| 단말 상태 | 조치 버튼 |
|-----------|-----------|
| 지속 장애 | 현장 점검 (보라색) |
| 장애 발생 | 재구동 + Ping (2개 버튼) |
| 수신 지연 | Ping 테스트 (노란색) |
| 정상 통신 | 상세 조회 (텍스트 링크) |

### 4. 영업 제안 발송 + 이력 자동 저장

각 데이터 화면에서 "대출 제안", "카드 제안", "유치 제안" 등 버튼 클릭 시 **제안 발송 모달**이 열립니다.  
발송 채널(이메일/SMS/카카오 알림톡) 다중 선택 및 AI 메시지 템플릿 수정이 가능합니다.

발송 완료 시 **제안 이력 관리** 화면에 자동으로 이력이 추가됩니다.

### 5. 자금흐름 고객사 필터 (드롭다운 + 검색 병행)

자금흐름 분석 화면에서 두 가지 방식으로 고객사를 선택할 수 있습니다.

- **드롭다운**: 전체 고객사 목록에서 바로 선택
- **검색창**: 고객사명 직접 입력 (자동완성 지원)
- 둘은 자동 동기화되며, 선택 시 KPI/차트/테이블이 실시간 전환됩니다.

## 파일별 역할

### db.js (586줄)

모든 데이터의 생성과 조회를 담당합니다.

- 14종 데이터 생성기 (`_genHosts`, `_genLogs`, `_genProposals`, `_genCampaigns`, `_genSegments` 등)
- 30+ 쿼리 메서드 (`getHostStats`, `getCashFlow`, `getProposalStats`, `getSegments` 등)

### dashboard.js (198줄)

시스템 관제 메인 대시보드를 렌더링합니다.

- KPI 5칸 (정상/주의/장애/가동률/지속장애)
- 장애 추이 라인차트 + 상태 도넛차트
- 금일 장애 / 반복 장애 / 최근 알람 3패널
- 마케팅 데이터 수집 현황
- VIP/일반/전체 필터

### control.js (435줄)

시스템 관제 하위 6개 화면을 렌더링합니다.

- 단말 상태 관제 (상태별 맞춤 조치 버튼)
- Flag 수신 이력 (유형별 카운트 + 필터)
- 재구동 이력 관리 (통계 + 테이블)
- 고객사 등급 관리, 운영 보고서, 모니터링 정책 설정

### data.js (411줄)

데이터 활용 8개 + 영업 자동화 3개 화면을 렌더링합니다.

- 데이터 활용 총괄 (종합 대시보드)
- 금융자산, 예적금, B2B, 법인카드, 세금계산서, 자금흐름 분석
- 제안 이력 관리 (KPI + 차트 + 필터 테이블)
- 캠페인 관리 (카드형 레이아웃 + 진행률)
- 고객 세그먼트 (좌측 목록 → 우측 상세)

### app.js (520줄)

앱 전체를 관리하는 코어입니다.

- SPA 라우팅 (`loadView` — 17개 뷰 등록)
- Customer 360 패널 (메뉴별 맞춤 분기)
- 모니터링 모달 (Agent 재구동 / Ping 테스트 동작)
- 영업 제안 발송 모달 (발송 시 DB 이력 자동 저장)
- Chart.js 인스턴스 관리

## 향후 확장 방향

현재 `db.js`의 인메모리 데이터를 실제 DB로 교체하는 것이 핵심입니다.

```
[현재 - 데모]
db.js (인메모리) → DB.getHosts() → UI 렌더링

[향후 - 운영]
MSSQL/Oracle → REST API → fetch('/api/hosts') → UI 렌더링
```

UI 코드(dashboard.js, control.js, data.js, app.js)는 수정 없이  
`db.js`의 각 메서드 내부만 API 호출로 바꾸면 됩니다.

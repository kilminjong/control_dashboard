# 하나은행 통합관제 및 데이터 대시보드 (CMS)

## 프로젝트 개요

하나은행 기업고객 대상 **CMS(Cash Management Service) 통합관제 시스템**입니다.  
고객사에 설치된 Agent 단말의 상태를 실시간으로 감시하고, 수집된 금융 데이터를 분석하여 영업/마케팅에 활용할 수 있는 올인원 대시보드입니다.

> 현재는 **데모(프로토타입)** 버전으로, 모든 데이터는 `db.js`에서 인메모리로 생성됩니다.  
> 향후 실제 MSSQL/Oracle 등 DB 연동 시 `db.js` 내부만 API 호출로 교체하면 됩니다.


---

## 폴더 구조

```
project/
├── index.html          ← 메인 HTML (사이드바, 헤더, 뷰 컨테이너)
└── js/
    ├── db.js           ← 데이터 레이어 (데이터 생성 + 쿼리 메서드)
    ├── dashboard.js    ← 시스템 관제 대시보드 총괄표
    ├── control.js      ← 시스템 관제 하위 화면들 (단말, Flag, 재구동 등)
    ├── data.js         ← 데이터 활용 화면들 (자산, B2B, 카드, 세금, 자금흐름)
    └── app.js          ← 앱 코어 (라우팅, 모달, Customer360, 초기화)
```


---

## 실행 방법

1. 위 폴더 구조 그대로 로컬에 저장합니다.
2. `index.html`을 브라우저에서 열면 바로 동작합니다.
3. 별도의 서버나 빌드 과정이 필요 없습니다. (CDN으로 Tailwind, Chart.js, FontAwesome 로드)

## 주요 기능

### 1. Customer 360 심층 분석 (메뉴별 맞춤)

어떤 화면에서든 **고객사 이름을 클릭**하면 오른쪽에서 심층 분석 패널이 슬라이드 됩니다.  
현재 보고 있는 메뉴에 따라 분석 내용이 달라집니다.

| 현재 메뉴 | 360 분석 내용 |
|-----------|--------------|
| 금융자산/예적금 | 은행별 보유 분포 + 만기 D-Day별 차트 + 보유 예적금 목록 |
| B2B/전자어음 | 만기 분포 + 거래유형별 비중 + 해당 고객 B2B 내역 |
| 법인카드 | 업종별 결제 비중 + 월별 추이 + 결제 내역 |
| 세금계산서 | 매출/매입 KPI + 품목별 분포 + 거래 내역 |
| 자금흐름 | 30일 입출금 차트 + 총입금/출금/순흐름 |
| 시스템 관제 | 인프라 레이더 + 60일 가동 히트맵 + 장애 추이 |

각 context별로 **AI 추천 액션**과 **제안 발송 버튼**도 맞춤 생성됩니다.

### 2. 모니터링 모달 (시스템 관제 전용)

단말 상태 관제 등에서 고객사 클릭 시 **Agent 단말 상세 정보 모달**이 열립니다.  
(IP, OS, 상태, 마지막 Ping, VIP 여부 등)

### 3. 영업 제안 발송 모달

각 데이터 화면에서 "대출 제안", "카드 제안", "유치 제안" 등 버튼 클릭 시  
**제안 발송 모달**이 열려 영업점/RM에게 전송할 수 있습니다.

### 4. 자금흐름 고객사별 필터

자금흐름 분석 화면에서 **고객사명을 검색 입력**하면 해당 고객사의 30일 입출금 데이터로  
KPI, 차트, 테이블이 모두 실시간 전환됩니다.

## 파일별 역할 상세

### db.js — 데이터 레이어

모든 데이터의 생성과 조회를 담당합니다. `DB` 객체 하나로 관리됩니다.

**생성되는 데이터:**

| 데이터 | 건수 | 설명 |
|--------|------|------|
| hosts | 300대 | Agent 단말 (정상/주의/장애/지속장애) |
| logs | 300건 | 시스템 로그 |
| flagMessages | 500건 | Flag 유형별 수신 이력 (6종) |
| rebootHistory | ~35건 | 재구동 시도 이력 |
| b2b | 300건 | 전자어음/B2B 거래 |
| tax | 300건 | 세금계산서 매출/매입 |
| assets | 150건 | 예적금 만기 |
| cards | 150건 | 타행 법인카드 결제 |
| cardMonthly | 32건 | 카드 월별 추이 (8개월 × 4업종) |
| depositDetail | 4건 | 은행별 예적금 잔액/만기/금리 |
| cashFlow | 450건 | 고객사별 30일 자금흐름 (15개사 × 30일) |

**주요 쿼리 메서드:**

- `getHostStats(filter)` — 단말 통계 (전체/VIP/일반)
- `getFlagCounts()` — Flag 유형별 집계
- `getRebootStats()` — 재구동 성공률/지속장애
- `getAssetsByCompany()` — 고객사별 예적금 집계
- `getCashFlow(company)` — 고객사별 자금흐름 (미지정 시 전체 합산)
- `getCashFlowRanking()` — 고객사 자금흐름 순위
- `getB2BSummaryByQuarter()` — B2B 분기별 집계
- `getTaxSummary()` — 세금계산서 매출/매입/우량 집계
- `getDepositSummary()` — 당행 점유율/총자산 집계
- `getRepeatOffenders(days)` — 반복 장애 고객사
- `getDataOverview()` — 데이터 활용 총괄 요약

### dashboard.js — 대시보드 총괄표

시스템 관제 메인 화면을 렌더링합니다.

- KPI 5칸 (정상/주의/장애/가동률/지속장애)
- 장애 추이 라인차트 + 상태 도넛차트
- 금일 장애 현황 / 반복 장애 고객 / 최근 알람 3패널
- 마케팅 데이터 수집 현황 4칸 KPI
- VIP/일반/전체 필터 전환

### control.js — 시스템 관제 하위 화면

- `renderPcStatus()` — 단말 상태 테이블 (persistent 상태 포함)
- `renderFlagHistory()` — Flag 유형별 카운트 카드 + 필터 + 테이블
- `renderRebootView()` — 재구동 이력 통계 + 테이블
- `renderGroupManage()` — 고객사 등급 관리
- `renderReportView()` — 운영 보고서
- `renderSettings()` — 모니터링 정책 설정

### data.js — 데이터 활용 화면

- `renderDataDashboard()` — 데이터 활용 총괄 (3개 차트 + 영업 타겟 + 자금흐름 순위)
- `renderAsset()` — 금융자산 점유 (스택 바차트 + 타겟 리스트)
- `renderDepositView()` — 예적금 분석 (도넛 + 은행별 만기 + 고객사별 테이블)
- `renderB2B()` — B2B 거래 (분기별 집계 + 테이블)
- `renderCard()` — 법인카드 (도넛 + 월별 추이 라인 + 테이블)
- `renderTax()` — 세금계산서 (KPI + 테이블)
- `renderCashFlow()` — 자금흐름 (고객사 검색 + 바+라인 차트 + 일별 테이블)

### app.js — 앱 코어

- `loadView()` — SPA 라우팅 (14개 뷰 등록)
- `openMonitoringModal()` — 단말 상세 모달
- `openCustomer360(name, context)` — 메뉴별 맞춤 심층 분석 패널
- `openProposalModal()` — 영업 제안 발송 모달
- `injectCustomer360Panel()` — 360 패널 HTML 주입
- `clearCharts()` — Chart.js 인스턴스 정리

## 향후 확장 방향

현재 `db.js`의 인메모리 데이터를 실제 DB로 교체하는 것이 핵심입니다.

```
[현재 - 데모]
db.js (인메모리) → DB.getHosts() → UI 렌더링

[향후 - 운영]
MSSQL/Oracle → REST API → fetch('/api/hosts') → UI 렌더링
```

UI 코드(dashboard.js, control.js, data.js, app.js)는 수정 없이  
`db.js`의 각 메서드 내부만 API 호출로 바꾸면 됩니다.
